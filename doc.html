Implementace

Projekt je implementován pomocí Laravel frameworku a zahrnuje aplikační vrstvu (PHP), databázovou vrstvu (SQL) a prezentační vrstvu (Blade šablony v HTML). Následuje stručný popis implementace jednotlivých klíčových use-casů a jejich propojení s částmi projektu.

Use-case scénáře a jejich implementace:

1. Uživatelé:

  • Model: User, Uzivatel (databázová tabulka Uzivatel)
  • Kontroler: UzivatelController, ProfileController, Auth::Controllery
  • Funkce: index(), create(), store(), validate(), show(), update(), destroy()
  • View:
    • uzivatele/index.blade.php - zobrazení seznamu uživatelů.
    • uzivatele/create.blade.php - vytvoření nového uživatele.
    • uzivatele/edit.blade.php - úprava konkrétního záznamu uživatele.
    • uzivatele/manage.blade.php - správa všech uživatelů.
    • uzivatele/show.blade.php - zobrazení konkrétního záznamu uživatele.
  • Role a autorizace: Jsou řešeny v rámci Auth kontrolerů. V ostatních kontrolerech je pozměněna funkcionalita pro jednotlivé role.

2. Moderátor spravuje kategorie plodin a atributů:

  • Model: Kategorie, Atribut (databázové tabulky Kategorie, Atribut)
  • Kontroler: UzivatelController
  • Funkce: index(), create(), store(), show(), update(), destroy()
  • View:
    • kategorie/index.blade.php - zobrazení seznamu kategorií.
    • kategorie/create.blade.php - vytvoření nové kategorie.
    • kategorie/edit.blade.php - úprava konkrétního záznamu kategorie.
    • kategorie/manage.blade.php - správa všech kategorií.
    • kategorie/show.blade.php - zobrazení konkrétního záznamu kategorie.
    • stejné views pro atributy

3. Registrovaný uživatel spravuje nabídky a samosběry:

  • Model: Nabidka, Relace_nabidka_atribut
  • Kontroler: NabidkaController
  • Funkce: index(), create(), store(), show(), update(), destroy()
  • View:
    • nabidky/index.blade.php - zobrazení seznamu všech relevantních nabídek pro danou roli.
    • nabidky/create.blade.php - vytvoření nové nabídky.
    • nabidky/edit.blade.php - úprava konkrétního záznamu nabídky.
    • nabidky/manage.blade.php - správa všech relevantních nabídek.
    • nabidky/show.blade.php - zobrazení konkrétního záznamu nabídky.

4. Zákazník provádí objednávky:

  • Model: Objednavka, Relace_objednavka_nabidka
  • Kontroler: ObjednavkaController
  • Funkce: index(), create(), store(), show(), update(), destroy(), updateForFarmer()
  • View:
    • objednavky/index.blade.php - zobrazení seznamu relevantních objednávek.
    • objednavky/create.blade.php - vytvoření nové objednávky.
    • objednavky/edit.blade.php - úprava konkrétního záznamu objednávky.
    • objednavky/manage.blade.php - správa všech objednávek.
    • objednavky/show.blade.php - zobrazení konkrétního záznamu objednávky.

6. Uživatel hodnotí produkty:

  • Model: Hodnoceni
  • Kontroler: HodnoceniController
  • Funkce: index(), create(), store(), show(), update(), destroy(),
  • View:
    • objednavky/index.blade.php - Seznam hodnocení.
    • objednavky/create.blade.php - vytvoření nového hodnocení.
    • objednavky/edit.blade.php - úpravování hodnocení.
    • objednavky/manage.blade.php - správa všech relevantních hodnocení, pro daného uživatele.
    • objednavky/show.blade.php - zobrazení konkrétního záznamu hodnocení.

Struktura databáze:

  • Databázový skript: DBserver.sql obsahuje definice tabulek a relací.
  • Klíčové tabulky: Uzivatel, Kategorie, Nabidka, Objednavka, Hodnoceni.

Poznámky:

  • Výjimečně je použit JavaScript pro interaktivní prvky na frontendu (např. filtry nebo potvrzení akcí).
  • Cesty a data mezi frontendem a backendem zajišťuje Laravel routing s podporou RESTful API.

Instalace

Postup instalace na server:
  • Prvním krokem byl import databáze pomocí rozhraní phpmyadmin na školním serveru.
  • Následně byl lokálně vygenerován aplikační klíč a upraven soubor .env pro nasazení projektu, společně s nastavením pro databázi.
  • Instalace na školní server proběhla pomocí zkomprimování repozitáře projektu bez složek node_modules, vendor a tests, a následného nahrání souboru na server.
  • Po rozbalení složky s projektem byl přesunut adresář public na požadované umístění a byl spuštěn příkaz php83 $(which composer) install --optimize-autoloader --no-dev.
  • Po úspěšné instalaci potřebných modulů byl projekt připraven a plně funkční.
Softwarové požadavky (verze PHP apod.):
  • Projekt vyžaduje pro inicializaci přítomnost PHP verze alespoň 8.2 a nainstalovaný Composer.
  • Pro inicializaci databáze doporučuji použít phpmyadmin.
  • Vše ostatní se nainstaluje během inicializace.
Jak rozbalit, konfigurovat, inicializovat databázi, ... (lokálně):
  • Rozbalení ZIP souboru.
  • Import databáze pomocí phpmyadmin.
  • Úprava souboru .env pro připojení k databázi.
  • Příkazem php composer install nainstalovat potřebné moduly.
  • Spustit server pomocí php artisan serve na localhostu.
